SYSTEM READY

Agents should not guess how your repo works. AgentML gives them a contract.

Diagnostic

Coding agents fail when project rules are invisible.

Three failure modes that appear in every agent-assisted codebase without a contract layer.

Unstructured

Instructions get ignored

README notes, chat history, and scattered rules are easy for agents to miss. Without enforcement, boundaries are suggestions.

Unsafe

Risky files need stronger boundaries

Auth, CI, dependency, release, and security files need explicit handling. A single blind edit can break a build or leak a secret.

Missing context

Agents lack repo awareness

A coding agent performs better when project rules, safe commands, sensitive paths, and skills are machine-readable and validated.

Protocol

AgentML converts repo intent into an agent-readable contract.

One file defines the rules. The CLI enforces them. The agent consumes structured context.

Features

Everything agents need to respect your repo.

⚬
Contract

Contract-first repo rules

Define what the project is, what commands are safe, what paths are sensitive, and what rules agents must respect.

permissions: read: ["**/*.ts"] validation: - cargo test
⊘
Paths

Sensitive path awareness

Declare the files agents must treat with extra care.

sensitive_paths: - src/auth/** - .github/workflows/** - Cargo.toml
⌘
Context

Generated agent context

Generate machine-readable context for any coding agent.

agentml context generates: .agentml/context.md
⚬
Skills

Reusable skill capsules

Define focused capabilities for validation, review, and spec writing.

skill: release-auditor actions: - check_changelog
⚠
Risk

Risk scoring before change

Surface risk around sensitive paths, destructive commands, and security files.

risk_score: 5 thresholds: - critical: 80
✓
CI

CI-enforced agent discipline

Run validation, skill checks, and self-checks in GitHub Actions on every push.

- uses: agentml/validate - uses: agentml/skill-check
